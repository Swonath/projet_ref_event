1. Présentation du Projet

En bref : Références Événements est une marketplace B2B qui connecte les locataires (entreprises cherchant des espaces temporaires) avec les centres commerciaux proposant des emplacements à la location.

Description

Références Événements est une plateforme web développée avec Symfony 7 qui permet la location d'emplacements commerciaux sur courte et moyenne durée. Elle fonctionne comme une marketplace mettant en relation deux types d'acteurs principaux :

  • Les locataires : particuliers ou entreprises cherchant à louer un emplacement temporaire (pop-up stores, stands d'exposition, kiosques, corners...)
  • Les centres commerciaux : gestionnaires d'espaces commerciaux qui proposent leurs emplacements disponibles à la location

Objectifs de la plateforme

  • Permettre aux centres commerciaux de publier et gérer leurs emplacements disponibles
  • Offrir aux locataires un moteur de recherche pour trouver l'emplacement idéal
  • Gérer le cycle complet des réservations (demande, validation, paiement, réalisation)
  • Faciliter la communication entre les deux parties via une messagerie intégrée
  • Automatiser la génération de documents (contrats, factures)
  • Permettre les évaluations post-location pour améliorer la confiance

2. Acteurs et Rôles

La plateforme utilise un système d'héritage de table (JOINED) pour gérer les différents types d'utilisateurs :

Locataire (ROLE_LOCATAIRE)

Profil type : Entrepreneur, commerçant temporaire, marque souhaitant tester un marché

Objectif : Trouver et réserver un emplacement commercial pour une durée déterminée

Informations stockées :

  • Nom, email, téléphone
  • SIRET (optionnel), type d'activité
  • Adresse de facturation, code postal, ville
  • Photo de profil, statut du compte

Centre Commercial (ROLE_CENTRE)

Profil type : Gestionnaire d'espace commercial, propriétaire de galerie marchande

Objectif : Rentabiliser ses espaces vacants en les proposant à la location temporaire

Informations stockées :

  • Nom du centre, SIRET, numéro TVA
  • Adresse complète, téléphone
  • Description, IBAN pour les paiements
  • Photo de profil, statut du compte

Administrateur (ROLE_ADMIN)

Profil type : Équipe de gestion de la plateforme

Objectif : Valider les inscriptions des centres, modérer les contenus, gérer les paramètres
